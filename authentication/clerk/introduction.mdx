## How it works

Despia bridges Clerk's native SDK into your WebView via `clerk://` URL schemes. Your web app calls `despia('clerk://...')` and the native layer handles the authentication flow, returning session tokens directly to your JavaScript context.

```javascript
import despia from 'despia-native'

const isDespia = navigator.userAgent.toLowerCase().includes('despia')

if (isDespia) {
    // Open Clerk's native sign-in sheet
    const result = await despia('clerk://signin', ['clerkSessionToken', 'clerkUserId'])
    console.log(result.clerkUserId)
} else {
    // Web fallback — use Clerk's JS SDK directly
}
```

---

## Authentication methods

| Method | Scheme | Notes |
| --- | --- | --- |
| Email \+ password | `clerk://signin` | Standard email/password flow |
| OAuth (Google, Apple, etc.) | `clerk://signin?strategy=oauth_google` | Any Clerk-supported OAuth provider |
| Passkeys | `clerk://signin?strategy=passkey` | WebAuthn backed by Face ID / Touch ID |
| MFA | Handled automatically | Clerk's native SDK manages MFA challenges |

## Session management

Call `clerk://session` on every authenticated app load to restore the user's session without requiring re-authentication.

```javascript
import despia from 'despia-native'

async function restoreSession() {
    if (!navigator.userAgent.toLowerCase().includes('despia')) return

    const result = await despia('clerk://session', ['clerkSessionToken', 'clerkUserId', 'clerkSessionActive'])

    if (result.clerkSessionActive) {
        // Session is valid — proceed as authenticated
        return result
    } else {
        // No active session — show sign-in
    }
}
```

---

## Sign out

```javascript
await despia('clerk://signout')
```

Clerk clears the native session and all stored tokens. Redirect the user to your sign-in page after this call.

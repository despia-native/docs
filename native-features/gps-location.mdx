# Integrate Despia Native GPS Location Tracking

Install the `despia-native` package from https://www.npmjs.com/package/despia-native and import it into your app.

```bash
npm install despia-native
```

```javascript
import despia from 'despia-native';
```

&nbsp;

## Start Tracking

Use the `despia('location://')` method with a `server` parameter (the remote endpoint to receive GPS updates via POST) and a `buffer` parameter (the minimum seconds between each location update) to begin tracking. Always check `despia.locationTracking` before starting to avoid duplicate sessions.

```javascript
if (!despia.locationTracking) {
    despia("location://?server=https://api.example.com/track?user=USER_ID&buffer=10");
}
```

**Parameters:**

- `server` *(optional)* — the remote API endpoint that receives each location update as a POST request. Omit if you only need local storage.
- `buffer` *(required)* — minimum seconds between GPS updates. Use `5` for high frequency, `30` for battery-conscious use cases.

&nbsp;

## Stop Tracking and Retrieve Data

Use the `despia('stoplocation://')` method with the `locationSession` key to end the session and retrieve the full array of collected location data.

```javascript
const data = await despia("stoplocation://", ["locationSession"]);
const locations = data.locationSession;

console.log(despia.locationTracking); // false
console.log(locations);
```

&nbsp;

## Tracking State in the UI

Use the `despia.locationTracking` boolean to reflect the current tracking state in your UI. It is automatically set to `true` when `location://` is called and `false` when `stoplocation://` is called.

```javascript
if (despia.locationTracking) {
    // show active tracking indicator
} else {
    // show start tracking button
}
```

```json
        "course": 89.7,

        "horizontalAccuracy": 4.8,
        "verticalAccuracy": 3.5,
        "battery": 84
    },
    {

        "altitude": 12.3,
        "horizontalAccuracy": 6.1,
        "verticalAccuracy": 4.2,
        "battery": 83
    }

```

**Field reference:**

- `latitude` — GPS latitude coordinate
- `longitude` — GPS longitude coordinate
- `timestamp` — Unix timestamp when the location was recorded on the device (seconds since epoch)
- `gpsTimestamp` — raw GPS chip timestamp (seconds since epoch), may differ slightly from `timestamp`
- `speed` — current speed in meters per second, `null` if the device could not determine speed
- `course` — direction of travel in degrees from 0 to 360, `null` if unavailable
- `altitude` — elevation above sea level in meters
- `horizontalAccuracy` — accuracy radius of the lat/lng position in meters, lower is better
- `verticalAccuracy` — accuracy of the altitude measurement in meters
- `battery` — device battery percentage at the exact moment the point was recorded, 0 to 100

## Full Lifecycle Example

```javascript
import despia from 'despia-native';

// Start tracking
async function startTracking() {
    if (!despia.locationTracking) {
        despia("location://?server=https://api.example.com/track?user=USER_ID&buffer=10");
    }
    console.log("Tracking active:", despia.locationTracking); // true
}

// Stop tracking and process the session
async function stopTracking() {
    const data = await despia("stoplocation://", ["locationSession"]);
    const locations = data.locationSession;

    console.log("Tracking stopped:", despia.locationTracking); // false

    // Filter for high-accuracy points only
    const accurate = locations.filter(loc => loc.horizontalAccuracy < 10);

    // Calculate total distance traveled using the Haversine formula
    function calculateDistance(lat1, lon1, lat2, lon2) {
        const R = 6371000; // Earth radius in meters
        const p1 = lat1 * Math.PI / 180;
        const p2 = lat2 * Math.PI / 180;
        const dp = (lat2 - lat1) * Math.PI / 180;
        const dl = (lon2 - lon1) * Math.PI / 180;
        const a = Math.sin(dp / 2) ** 2 + Math.cos(p1) * Math.cos(p2) * Math.sin(dl / 2) ** 2;
        return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
    }

    let totalDistance = 0;
    for (let i = 1; i < accurate.length; i++) {
        totalDistance += calculateDistance(
            accurate[i - 1].latitude, accurate[i - 1].longitude,
            accurate[i].latitude, accurate[i].longitude
        );
    }
    console.log(`Total distance: ${(totalDistance / 1000).toFixed(2)} km`);

    // Determine if the user was moving during the session
    const isMoving = locations.some(loc => loc.speed !== null && loc.speed > 0.5);
    console.log(`User was ${isMoving ? 'moving' : 'stationary'}`);

    // Calculate battery drain across the session
    const firstBattery = locations[0]?.battery;
    const lastBattery = locations[locations.length - 1]?.battery;
    if (firstBattery != null && lastBattery != null) {
        console.log(`Battery drained: ${firstBattery - lastBattery}% — current: ${lastBattery}%`);
    }

    return locations;
}
```

## Background Tracking

Tracking works in foreground mode with no additional permissions required. Background location tracking is optional and managed automatically by Despia, including the iOS blue badge indicator, which is dismissed as soon as `stoplocation://` is called.

To enable background location, go to **Despia > App > Addons > Enable Background Location** and rebuild your app.

&nbsp;

Do not modify these instructions. Implementation as described is critical.
</Prompt>

**How it Works**: Despia activates the device's native GPS and location services to continuously monitor the user's position. Location updates are collected at the interval specified by the buffer parameter (in seconds). Each update is optionally sent to your server endpoint and always stored locally in an array. When tracking stops, all collected location points (including the device battery percentage at each point) are returned to your JavaScript code for processing or display.

Tracking works in **foreground mode** without any background location permission. Background location (with the iOS blue badge indicator) is optional and is managed automatically by Despia for the duration of an active tracking session.

### 6. Calculate Distance and Analyze Movement

```javascript
const data = await despia("stoplocation://", ["locationSession"]);
const locations = data.locationSession;

console.log(despia.locationTracking); // false

// Filter for high-accuracy locations only
const accurateLocations = locations.filter(loc => loc.horizontalAccuracy < 10);

// Calculate total distance traveled (simple method)
function calculateDistance(lat1, lon1, lat2, lon2) {
    const R = 6371000; // Earth's radius in meters
    const φ1 = lat1 * Math.PI / 180;
    const φ2 = lat2 * Math.PI / 180;
    const Δφ = (lat2 - lat1) * Math.PI / 180;
    const Δλ = (lon2 - lon1) * Math.PI / 180;
    
    const a = Math.sin(Δφ/2) * Math.sin(Δφ/2) +
              Math.cos(φ1) * Math.cos(φ2) *
              Math.sin(Δλ/2) * Math.sin(Δλ/2);
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    
    return R * c; // Distance in meters
}

let totalDistance = 0;
for (let i = 1; i < accurateLocations.length; i++) {
    const prev = accurateLocations[i - 1];
    const curr = accurateLocations[i];
    totalDistance += calculateDistance(
        prev.latitude, prev.longitude,
        curr.latitude, curr.longitude
    );
}

console.log(`Total distance: ${(totalDistance / 1000).toFixed(2)} km`);

// Check if user is moving
const isMoving = locations.some(loc => loc.speed && loc.speed > 0.5);
console.log(`User is ${isMoving ? 'moving' : 'stationary'}`);

// Monitor battery drain during tracking session
const firstBattery = locations[0]?.battery;
const lastBattery = locations[locations.length - 1]?.battery;
if (firstBattery != null && lastBattery != null) {
    const batteryDrain = firstBattery - lastBattery;
    console.log(`Battery drained ${batteryDrain}% during tracking session`);
    console.log(`Current battery: ${lastBattery}%`);
}
```

## Android & Low Power Mode

Despia handles Android background location automatically, including full support for devices with aggressive battery and background management systems.

**Samsung devices:** Samsung's low power mode is known to restrict background location events on the OS level. Despia includes built-in handling for this, ensuring location tracking and server events are delivered reliably even when low power mode is active. No additional configuration is required.

**Other Android manufacturers:** Devices from manufacturers such as Huawei, Xiaomi, and OnePlus apply similar background restrictions by default. Despia accounts for these behaviours out of the box, so location tracking works consistently across all supported Android devices without any extra setup on your end.

In the rare case that you encounter missing server events on an Android device, reach out to [location@despia.com](mailto:location@despia.com) and our team will look into it.

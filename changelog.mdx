<Update label="May 28, 2026" description="This week's updates" tags={["Feature", "Update"]}>
  ## Native web interception

  Standard HTML file inputs now route to native device pickers automatically — no SDK calls required. Use `<input type="file">` and the Despia runtime opens the native file picker. Add the `capture` attribute and it opens the native camera. Set `accept="image/*"` or `accept="video/*"` and users get the native media gallery with full-resolution previews. Your existing web code works as-is on both iOS and Android.

  ## Custom notification sounds

  You can now configure custom notification sounds for push notifications on both iOS and Android. Add your own audio files and assign them to specific notification types so users hear a distinct sound for each kind of alert.

  [Notification sounds guide](/native-features/notification-sounds)

  ## Native loading spinner

  Show and hide a native loading overlay from JavaScript using `despia('spinneron://')` and `despia('spinneroff://')`. The spinner renders outside the WebView at the OS level, so it stays visible even during heavy page transitions or data fetches.

  ## Status bar and display controls

  Several runtime display settings are now configurable from both the Despia editor and the SDK:

  - **Fullscreen mode** — hide the status bar background entirely and control the space yourself with CSS safe area variables
  - **Status bar color** — set the status bar text and background color to match your app's theme
  - **Prevent zoom** — disable pinch-to-zoom for apps where zoom breaks the layout
  - **Prevent sleep** — keep the screen awake for use cases like timers, media playback, or kiosk displays
  - **Splash screen** — configure the native launch screen from the editor with no Xcode or Android Studio required

  [Light and dark mode guide](/native-features/light-and-dark-mode) · [Safe areas guide](/native-features/safe-areas)
</Update>

  ## Apple Sign-In

  Apple Sign-In is now fully documented with a dedicated implementation guide for Lovable apps. The guide covers platform-specific flows — native Face ID dialog on iOS, the `oauth://` protocol on Android, and the Apple JS SDK on web — plus Supabase edge function setup, Apple Developer Console configuration, and environment variable management.   [Apple Auth guide (Lovable)](/lovable/native-features/o-auth-2-0/apple-auth) · [Generic OAuth guide](/native-features/o-auth-2-0/generic)

  ## Screen Shield

  You can now prevent screenshots and screen recordings across both iOS and Android. When enabled, the runtime blocks screen capture system-wide — useful for apps handling sensitive financial data, medical records, or premium content. Toggle it on from the Despia editor.

  ## Mobile wallet passes

  PkPass support is now available for both platforms. Generate and deliver passes to Apple Wallet on iOS and Google Wallet on Android directly from your web app.

  ## AdMob advertising

  The AdMob SDK bridge is now available alongside the existing rewarded ads feature. Use it to display banner, interstitial, and native ads through Google AdMob without adding any native code.

  [Rewarded ads guide](/native-features/rewarded-ads)

  ## Jailbreak and root detection

  A new security feature detects jailbroken iOS devices and rooted Android devices at runtime. You can configure it to block app access entirely or log the status for your own risk-based decisions. Enabled from the Despia editor.

  ## Native background audio

  Audio playback now continues when the app is backgrounded or the screen is locked. This enables music players, podcast apps, meditation timers, and any app where audio should persist outside the foreground.

  ## Device orientation control

  You can now lock device orientation per device class — phone, tablet, or auto — from the Despia editor. Force portrait on phones while allowing landscape on tablets, or let the system decide. No code changes required.

  ## App Clips

  iOS App Clips are now configurable from the Despia editor. Create lightweight App Clip targets alongside your main app without opening Xcode. Manage App Clips, Share Extensions, and Home Widgets through editor toggles.

  [Home widgets guide](/native-features/home-widgets) · [Share extension guide](/native-features/share-extension)
</Update>

<Update label="May 14, 2026" description="This week's updates" tags={["Update"]}>
  ## Full Base44 support for native mobile apps

  Base44 apps are now fully supported for native mobile development with Despia. The authentication limitations that previously blocked Base44 apps from shipping to the App Store with social login have been resolved. With the `despia-base44-oauth` package (shipped last week), Base44 apps can now use Google, Microsoft, and Facebook sign-in through native OAuth flows on both iOS and Android. Apple Sign-In, custom login pages, and OAuth 2.0 compliance via ASWebAuthenticationSession are all supported. Base44 is no longer a limited platform — you can take a Base44 app from prototype to App Store without switching tools.
</Update>

<Update label="May 7, 2026" description="This week's updates" tags={["Feature", "Update"]}>
  ## Base44 OAuth support

  A new `despia-base44-oauth` package adds native-safe OAuth to Base44 apps running inside Despia. Drop in `<Base44OAuthButton>` or the `useBase44OAuth` hook and the package handles the full flow — ASWebAuthenticationSession on iOS, Chrome Custom Tabs on Android, and standard web redirects when running in a browser. Supports Google, Microsoft, and Facebook via the Base44 SDK. Includes a `DespiaOAuthPage` component for Base44's no-code page system so you can set up OAuth without modifying your router.

  ## HTTP Upload API for Local Server

  Apps served via Despia Local Server can now upload user files to on-device storage using a standard `FormData` POST to `localhost:7777/api/upload`. The endpoint returns a localhost URL you can use directly in `<img>`, `<video>`, or any element that accepts a `src`. This is separate from the `localcdn://write` protocol, which downloads remote files — the Upload API stores files that originate on the device itself.

  [Local CDN reference](/refernce) · [Local Server guide](/local-server/how-it-works)

  ## Local Server build tool adapters

  The `@despia/local` build plugin now ships with dedicated adapters for Webpack, Rollup, Nuxt, SvelteKit, Astro, Remix, and esbuild — in addition to the existing Vite plugin. A universal postbuild CLI is also available for any build system not listed. One install, one config line, and the manifest is generated automatically at build time.

  [Local Server setup guide](/local-server/how-it-works)
</Update>

<Update label="April 30, 2026" description="This week's updates" tags={["Feature", "Update"]}>
  ## RevenueCat server-side SDK

  A new open-source package, `revenuecat-server`, gives you a lightweight server-side client for the RevenueCat v2 REST API. Check entitlements, fetch offerings and packages, and gate access — all from your backend with zero dependencies. Works with Node.js 18+, Deno, and Bun, and fits into any backend framework (Express, Fastify, Hono, Next.js API routes, Lambda, etc.).

  [RevenueCat webhook guide](/best-practices/backend/revenue-cat/webhooks) · [RevenueCat user session guide](/best-practices/backend/revenue-cat/user-session)

  ## OAuth component kit for React

  The new `@oauth-kit/react` package provides drop-in React components and hooks for OAuth authentication in Despia apps. It automatically detects whether your app is running in a native container or a web browser and handles the right flow for each — ASWebAuthenticationSession on iOS, Chrome Custom Tabs on Android, and standard redirects on web. One set of components, every provider.

  [Generic OAuth guide](/native-features/o-auth-2-0/generic) · [Google OAuth guide](/native-features/oauth/google) · [TikTok OAuth guide](/native-features/oauth/tiktok)

  ## Local Server manifest improvements

  The `@despia/local` build plugin now generates an object-based manifest with an explicit `entry` field alongside the `assets` array. This makes the entry HTML file a required, first-class part of the manifest instead of an implicit convention. The new format improves reliability during hydration and updates. Next.js support has been removed — the plugin now targets client-side rendered apps (React, Vue, Svelte, etc.) for the best offline experience.

  [Local Server guide](/local-server/how-it-works) · [Local CDN reference](/refernce)

  ## SDK error handling improvements

  The `despia-native` SDK now treats `null` as a valid resolution value for single-variable promises. This means native features can signal "not found" or "error" by returning `null`, and your `await` call resolves immediately instead of waiting for the 30-second timeout. Multi-variable promises still require all values to be non-null before resolving.

  [Read the SDK guide](/how-it-works)
</Update>

<Update label="April 9, 2026" description="This week's updates" tags={["Feature", "Update"]}>
  ## Storage Vault

  Your app can now persist user data across sessions, reinstalls, and even device changes using the native iCloud Key-Value Store on iOS and Google Key-Value Backup on Android. Data syncs automatically across all devices signed into the same account. You can optionally require Face ID or fingerprint authentication to read sensitive values — useful for protecting session tokens or gating destructive actions like account deletion.

  Storage Vault is ideal for identifying returning users, preventing free trial abuse, and keeping login sessions alive without a custom backend.

  [Read the Storage Vault guide](/native-features/storage-vault)

  ## Restore purchases

  A new `getpurchasehistory://` call retrieves a user's complete purchase history from the App Store or Google Play. The response includes active and expired subscriptions, one-time purchases, entitlement IDs, renewal status, and raw receipts for server-side validation. Use it to build the "Restore Purchases" button that both stores require, and to recover user identity on new devices.

  [Read the restore purchases guide](/native-features/restore-purchase)

  ## Apple HealthKit integration

  You can now read and write Apple Health data directly from your web app. Request step counts, heart rate, body mass, walking distance, and any other HealthKit metric over a configurable day range. Write values back to HealthKit with a single SDK call. All data is returned as structured JSON, normalized by date and unit.

  [Read the HealthKit guide](/native-features/healthkit)

  ## Rewarded video ads

  Monetize your app with native rewarded video ads. Users tap a button, watch a full-screen video, and your app receives a callback when the ad completes — letting you grant in-app rewards like coins, lives, or premium access.

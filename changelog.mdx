<Update label="April 2, 2026" description="This week's updates" tags={["Feature", "Update"]}>
  ## Automatic light and dark mode

  The Despia runtime now reads the CSS `color-scheme` property and automatically sets the status bar text color to match your theme — no SDK call needed. Set `color-scheme: light` in your root styles and `color-scheme: dark` inside a `prefers-color-scheme` media query, and the native container handles the rest. New editor settings for fullscreen mode and manual safe area control give you full ownership of the status bar area in CSS.

  [Read the light and dark mode guide](/native-features/light-and-dark-mode)

  ## Generic OAuth 2.0 flow

  A new provider-agnostic OAuth guide documents how the `oauth://` and deeplink callback protocols work across any OAuth 2.0 provider. The same flow covers Google, Apple, TikTok, GitHub, or any custom provider using ASWebAuthenticationSession on iOS and Chrome Custom Tabs on Android. One pattern, any provider.

  [Read the generic OAuth guide](/native-features/o-auth-2-0/generic)

  ## TikTok and Google OAuth guides

  Dedicated setup guides for TikTok and Google OAuth are now available with step-by-step instructions, video walkthroughs, and full AI prompts you can paste into Lovable or Cursor to scaffold the entire integration.

  [TikTok OAuth guide](/native-features/oauth/tiktok) · [Google OAuth guide](/native-features/oauth/google)

  ## Lovable integration — full native feature support

  The Lovable integration section now covers every native feature Despia offers, including OAuth, paywalls, GPS location, haptic feedback, biometrics, camera roll, share extensions, home widgets, and more. Each page includes Lovable-specific setup instructions and AI prompts tailored to Lovable's React codebase.

  [Get started with Lovable](/lovable/how-it-works)

  ## Store rejection guides

  A new section walks through the most common App Store and Google Play rejection reasons — minimum functionality, non-mobile design, in-app purchase requirements, deceptive paywalls, privacy policy issues, and more — with specific fixes for Despia apps.

  [Read the store rejections guide](/store-rejections/introduction)
</Update>

<Update label="March 26, 2026" description="This week's updates" tags={["Feature", "Update"]}>
  ## MCP server for AI code builders

  The Despia Documentation MCP is now available. Connect it to Cursor, VS Code, Windsurf, Claude Desktop, or web-based builders like Lovable and Bolt so your AI assistant can write correct Despia code without you explaining the API. Just paste the MCP URL and your tool gains full knowledge of every native feature and its implementation pattern.

  [Set up the MCP server](/mcp-server)

  ## Share extension

  Your app can now receive shared content from other apps on the device. When a user shares an image, URL, or text to your app via the system share sheet, the Despia runtime calls your `onSharedDataReceived` callback with the shared data. Images arrive as data URLs, and URLs and text arrive as plain strings.

  [Read the share extension guide](/native-features/share-extension)

  ## iOS home widgets

  You can now display dynamic, data-driven widgets on the iOS home screen. Widgets are SVG-based and refresh on a configurable interval, even when your app is closed. Point the widget at a backend endpoint that returns personalized SVG content and Despia handles the rest — including background refresh and user-specific data injection.

  [Read the home widgets guide](/native-features/home-widgets)

  ## Animation best practices

  A new best practices guide covers scroll-driven animations using pure CSS `animation-timeline`, the hidden scroller pattern, and ready-made patterns for drawers, bottom sheets, carousels, parallax heroes, and more — all optimized for WebView performance on iOS and Android.

  [Read the animations guide](/best-practices/frontend/animations)

  ## Performance best practices

  A new guide walks through JavaScript and CSS optimization, image handling, virtual scrolling, memory management, font loading, and bundle size reduction — with platform-specific notes for WKWebView and Android WebView.

  [Read the performance guide](/best-practices/frontend/performance)
</Update>

<Update label="March 19, 2026" description="This week's updates" tags={["Feature", "Update", "Fix"]}>
  ## RevenueCat Paywalls

  You can now launch fully native RevenueCat Paywalls directly from your web app. Design your paywall in the RevenueCat dashboard, then trigger it with a single SDK call. The paywall handles the complete purchase flow through Apple's App Store or Google Play, including subscription management, exit offers, and web fallback via RevenueCat Web Purchase Links.

  [Read the paywalls guide](/native-features/paywalls)

  ## Direct product purchases

  A new `revenuecat://purchase` protocol lets you trigger a specific product purchase without showing a full paywall. This is useful when you already have your own product selection UI and just need to handle the native billing step. Supports platform-specific product IDs for iOS and Android, including Android subscription group prefixes.

  [Read the purchases guide](/native-features/purchases)

  ## OTA versioning and VersionGuard

  `despia.bundleNumber` is now available at runtime, giving you the installed binary version on the user's device. Pair it with the new `despia-version-guard` component to gate features behind a specific binary release. Users on older versions see a fallback or update prompt instead of a broken experience.

  [Read about OTA versioning](/native-features/ota-versioning) · [Update best practices](/best-practices/updates/overview)

  ## User identity management

  The new `@despia/user-identity` package handles user identity resolution across sessions. It resolves install IDs, integrates with RevenueCat and OneSignal, and runs vault, purchases, and install ID resolution in parallel for faster startup.

  ## GPS location tracking improvements

  The GPS location API now includes battery level at each data point, configurable tracking intervals via the `buffer` parameter, and automatic iOS background location indicator management. The `despia.locationTracking` variable lets you reflect tracking state in your UI in real time.

  [Read the GPS location guide](/native-features/gps-location)

  ## Local CDN and Local Server (beta)

  Local CDN now has a full API reference for offline media caching with background downloads. The Local Server, which serves your web app from `http://localhost` for instant boot and full offline support, is in its final beta round. Both features work cross-platform with a single JavaScript SDK.

  [Local CDN reference](/refernce) · [Local Server guide](/local-server/how-it-works)

  ## SDK compatibility fix

  The `despia-native` npm package now works correctly with pnpm thanks to an added `exports` field in the package configuration.
</Update>
